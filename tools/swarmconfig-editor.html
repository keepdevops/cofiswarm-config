        Schema cheat sheet

        Root: object with agents (array). Optional coordinator, ui, comments keys like _ram_tier ignored by runtime.

        Each agent: name, model (file or MLX dir), context, read_timeout_secs, max_tokens, system_prompt; optional port, server_group, gpu_layers, backend/engine. Backend defaults: .gguf → llama inference stack.

        Shared port: same backend/engine + port allowed when server_group and model match (llama-server grouping).

        coordinator.modes.pipeline: order: string[] agent names; optional synthesizer string.

        coordinator.modes.router: classifier, choices, fallback (string arrays/names), max_select number.

        Deploy: proxy reads config for launch grouping — see src2/proxy_configure.cpp. Coordinator loads agents from JSON — see src2/coordinator.cpp.

        Merge JSON

        Paste a full or partial swarm JSON. Agents merge by name (incoming fields overlay existing). coordinator and ui are deep-merged; other top-level keys prefer incoming.
